I started False Systems as a joke. Now it's a serious attempt to
figure out what infrastructure looks like when agents are the operators.

I build tools that are transparent and easy to maintain — not because
it's elegant, but because the next person (human or agent) needs to
understand what they're touching.

---

### false systems

Infrastructure that assumes agents operate it.

**observe**
- [`ahti`](https://github.com/yairfalse/ahti) — causal graph for infrastructure state `rust`

**act**
- [`sykli`](https://github.com/yairfalse/sykli) — ci without yaml `elixir`
- [`nopea`](https://github.com/false-systems/nopea) — cd that remembers what it deployed `go`
- [`rauha`](https://github.com/yairfalse/rauha) — safe boundaries for agents that act `rust`
- [`rauta`](https://github.com/yairfalse/rauta) — gateway api proxy, from scratch `rust`

**connect**
- [`polku`](https://github.com/false-systems/polku) — typed event routing for agents `go`
- [`kerto`](https://github.com/false-systems/kerto) — context that survives the session `go`

**foundations**
- [`vaisto`](https://github.com/false-systems/vaisto) — scheme with hindley-milner, for the beam `elixir`
- [`vaisto_bpf`](https://github.com/yairfalse/vaisto_bpf) — compile typed s-expressions to bpf bytecode `elixir`
- [`ilmari`](https://github.com/yairfalse/ilmari) — go kubernetes sdk, no config files `go`
- [`seppo`](https://github.com/yairfalse/seppo) — rust kubernetes sdk `rust`
